# Vehicle Memory Bank Benchmark

Система поиска похожих автомобилей по текстовому описанию, метаданным и визуальным признакам. Решение предназначено для задач поиска объектов в каталоге, подбора похожих ассетов и проверки гипотез по многомодальному поиску.

## Назначение

По описанию автомобиля и связанным признакам находить наиболее релевантные объекты в каталоге и удерживать корректные совпадения в верхних позициях выдачи.

Типовые сценарии:

- поиск похожих автомобилей в memory bank;
- подбор ассетов для редактирования сцены;
- дедупликация каталога;
- сравнение конфигураций поиска на компактном воспроизводимом наборе.

## Архитектура

```text
Каталог автомобилей
        │
        ▼
Подготовка признаков
 ├─ текстовые признаки
 ├─ табличные признаки
 └─ визуальные признаки
        │
        ▼
Объединение сигналов
        │
        ▼
Ранжирование
        │
        ├─ расчёт метрик качества
        └─ API поиска
```

## Что реализовано

- загрузка локального каталога автомобилей из CSV;
- индексирование текстовых описаний ракурсов;
- учёт совпадений по марке, модели, типу кузова и цвету;
- извлечение компактных визуальных признаков для изображений;
- объединение нескольких источников сигнала при ранжировании;
- расчёт метрик качества на задаче поиска другого ракурса того же автомобиля;
- API для поиска похожих объектов.

## Структура репозитория

- `configs/` — конфигурации запуска;
- `data/demo_catalog/` — демонстрационный каталог автомобилей;
- `docs/` — материалы по архитектуре;
- `examples/` — примеры запросов;
- `scripts/` — сценарии запуска оценки и API;
- `src/vehicle_bank/` — основной код поиска, ранжирования и оценки;
- `tests/` — тесты;
- `Dockerfile`, `Makefile` — воспроизводимый запуск.

## Оценка качества

Основные метрики:

- `Recall@1`
- `Recall@3`
- `MRR`
- `top1_brand_match`
- `top1_color_match`
- задержка ответа: `latency_ms_p50`, `latency_ms_p95`

Сравнение ведётся как минимум для двух режимов:

- поиск по тексту и метаданным;
- объединённый режим с учётом визуальных признаков.

## Результаты

|--------|----------:|

| MRR | 1.0 |
| top1_brand_match | 1.0 |
| top1_color_match | 1.0 |

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python scripts/run_benchmark.py --config configs/demo_text.yaml
python scripts/run_benchmark.py --config configs/demo_hybrid.yaml

python scripts/serve_api.py
```

## Ограничения

- качество визуальных признаков зависит от однородности изображений;
- небольшие каталоги чувствительны к составу тестового набора;
- для промышленного применения потребуется расширение каталога и формальное сравнение нескольких схем объединения сигналов.
