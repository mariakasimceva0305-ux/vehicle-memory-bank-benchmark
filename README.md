# Vehicle Memory Bank Benchmark

Система поиска похожих автомобилей по тексту, метаданным и визуальным признакам. Решение сочетает несколько источников сигнала и предоставляет API для запроса похожих объектов.

## Задача
Нужно по описанию автомобиля и связанным признакам находить наиболее похожие объекты в каталоге, удерживая релевантные совпадения в верхних позициях выдачи.

## Архитектура решения
```text
Каталог автомобилей
        │
        ▼
Подготовка признаков
 ┌────────────┼────────────┐
 ▼            ▼            ▼
Текст     Метаданные   Визуальные признаки
 └────────────┼────────────┘
              ▼
        Объединение сигналов
              │
              ▼
       Ранжирование кандидатов
              │
              ├──► Оценка качества
              └──► API
```

## Структура репозитория
- `configs/` — параметры запуска;
- `data/demo_catalog/` — демонстрационный каталог;
- `src/vehicle_bank/` — основной код;
- `scripts/` — запуск оценки и сервиса;
- `examples/` — примеры использования;
- `docs/` — дополнительные материалы;
- `tests/` — тесты;
- `Dockerfile`, `Makefile` — воспроизводимый запуск.

## Метрики
- `Recall@1`
- `Recall@3`
- `MRR`
- `top1_brand_match`
- `top1_color_match`
- задержка ответа (`latency_ms_p50`, `latency_ms_p95`)

## Результаты
В финальной версии README должны быть указаны реальные значения по каждому режиму поиска. Необходимый минимум:
- сравнение текстового поиска, поиска по метаданным, поиска с визуальными признаками и объединённого варианта;
- значения `Recall@1`, `Recall@3`, `MRR`;
- доля случаев, где в первой позиции совпали марка и цвет;
- задержка `p50` и `p95`.

Формат записи:
- `Объединение текстовых, табличных и визуальных признаков повысило Recall@3 с ... до ...`
- `Совпадение марки в первой позиции выросло с ... до ...`
- `Задержка p95 составила ... мс`

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scripts/run_benchmark.py --config configs/demo_hybrid.yaml
python scripts/serve_api.py
```

## Ограничения и развитие
- Визуальные признаки чувствительны к качеству изображений;
- качество метаданных влияет на итоговую устойчивость выдачи;
- следующий шаг — сохранять итоговые таблицы качества в `docs/` или `reports/`.
