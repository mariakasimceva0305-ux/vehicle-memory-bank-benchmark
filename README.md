# Vehicle Memory Bank Benchmark

## Кратко
Стенд сравнения поиска похожих автомобилей по нескольким сигналам: текстовому описанию, метаданным и визуальным признакам. Репозиторий позволяет сравнивать базовый текстовый режим с улучшенными конфигурациями и оценивать выигрыш по качеству поиска и по задержке ответа.

## Задача
Нужно находить в базе автомобили, максимально похожие на входной объект. Один только текст не всегда даёт достаточную точность: у кратких описаний может не хватать деталей, а отдельные признаки удобнее извлекать из структуры карточки и визуальных характеристик. Поэтому поиск строится как комбинация нескольких сигналов.

## Что улучшено
- по сравнению с текстовым базовым режимом добавлены структурированные признаки объекта;
- добавлены визуальные признаки, которые помогают различать близкие по описанию варианты;
- повышена устойчивость поиска на случаях, где текст краткий или неполный;
- улучшение оценивается не только по качеству совпадения, но и по задержке ответа.

## Архитектура
```text
[входной объект]
      |
      +-------------------+
      |                   |
      v                   v
[текстовые признаки] [метаданные]
      |                   |
      +---------+---------+
                |
                v
      [визуальные признаки]
                |
                v
   [объединение сигналов и поиск]
                |
                v
     [ранжированный список похожих
              автомобилей]
                |
                v
      [API на FastAPI и оценка качества]
```

## Структура репозитория
```text
configs/              конфигурации экспериментов
data/demo_catalog/    демонстрационные данные
docs/                 документация
examples/             примеры запросов
scripts/              вспомогательные сценарии
src/vehicle_bank/     основная логика поиска и API
tests/                тесты
```

## Критерии качества
- Recall@1
- Recall@3
- MRR
- точность совпадения по марке в верхней позиции
- точность совпадения по цвету в верхней позиции
- задержка ответа

## Ключевой вывод
Добавление метаданных и визуальных признаков к текстовому поиску повышает точность верхних позиций выдачи и помогает лучше различать близкие объекты. Основной практический компромисс — рост вычислительной нагрузки, поэтому в итоговой конфигурации важно балансировать прирост качества и задержку ответа.

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn vehicle_bank.api:app --reload
```

## Ограничения
- качество визуальных признаков зависит от однородности изображений;
- на очень малом объёме данных выигрыш от сложной многосигнальной схемы может быть ограничен;
- итоговая конфигурация должна подбираться с учётом требований по задержке.
